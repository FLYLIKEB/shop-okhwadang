# 자사몰 전환 배경

### 1. 스마트스토어 운영 비용 구조

다음과 같은 비용·제약이 있습니다.

- **수수료 독점:** 매출의 2~3.5%를 평생 지불. 자사호는 객단가가 높은 만큼 건당 수수료 손실도 큽니다.
- **디자인 제약:** 수십만 개 셀러와 동일한 템플릿 안에서, 자사호의 품격과 작가·산지의 이야기를 전달하기 어렵습니다.
- **데이터 제약:** 재구매율이 높은 다구 고객임에도, 내 고객 데이터를 마케팅에 자유롭게 활용할 수 없습니다.

### 2. 매출 구간별 수수료 절감 시뮬레이션

| 월 거래액 | 플랫폼 수수료 (연간) | 자사몰 서버비 (연간) | **연간 절감액** | 개발비 회수 시점 |
|----------|------------------|------------------|--------------|----------------|
| 3,000만 원 | 720~1,260만 원 | 60~120만 원 | **660~1,140만 원** | 약 1년 내 |
| 5,000만 원 | 1,200~2,100만 원 | 60~120만 원 | **1,140~1,980만 원** | 약 6개월 내 |
| 1억 원 | 2,400~4,200만 원 | 60~120만 원 | **2,280~4,080만 원** | 약 3개월 내 |

### 3. 자사몰 주요 기능

| 핵심 가치 | 상세 설명 |
|----------|----------|
| **다구 전문 카테고리** | 자사호 · 보이차 · 다구 세트 · 산지별 찻잎 등 옥화당 특화 카테고리 구조 설계 |
| **작가·산지 스토리텔링** | 각 자사호의 작가 소개, 산지·흙 종류·제작 과정을 담는 전용 스토리 페이지 |
| **운영의 자유** | 개발자 없이 어드민에서 블록(섹션)을 조립해 메인 페이지를 실시간 변경 |
| **기술적 독립** | PG사·배송사를 필요에 따라 어드민 설정만으로 교체 가능한 유연한 구조 |
| **검색 노출** | SSR + JSON-LD 구조화 데이터로 구글·네이버 상단 노출 유도 |
| **고성능 인프라** | Vercel(프론트) + AWS EC2(백엔드) + Lightsail MySQL 검증된 배포 구조 |
| **글로벌 스토어** | 한국어/영어 다국어 지원, 한국가격/미국가격 개별 설정 가능 |
| **네이버페이 연동** | 네이버페이 결제 필수 적용 |
| **스마트스토어 리뷰 연동** | 네이버 스마트스토어 리뷰를 자사몰에 표시 (상품은 자사몰에서 별도 관리) |
| **인스타그램 쇼핑** | Instagram Shopping 연동으로 SNS 판매 채널 확장 |

### 4. 개발 일정 (8주)

| 주차 | 내용 | 산출물 |
|------|------|--------|
| 1~2주차 | UI 시안 3개 병렬 제작 → 고객 선택 후 확정 | 디자인 시안 3종, 확정 시안 |
| 3~5주차 | 핵심 상거래 엔진 + 결제(네이버페이 포함) + 글로벌 설정 | 중간 데모 사이트 |
| 6~7주차 | 어드민(쿠폰/메시지 발송/가격 관리) + 외부 연동(인스타/스마트스토어 리뷰) | 어드민 데모 |
| 8주차 | 전 기기 QA + 스마트스토어 데이터 이전 + 런칭 | 최종 납품 |

납품 후 1개월 무상 버그 수정 포함.
