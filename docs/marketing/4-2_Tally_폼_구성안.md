# 옥화당 자사몰 구축 사양 — Tally.so 폼 구성안

> 체크리스트(`4-1_구축_사양_체크리스트.md`)를 Tally.so 폼으로 변환하기 위한 설계 문서입니다.
> 고객(옥화당 대표님)이 모바일에서 편하게 입력할 수 있도록 설계되었습니다.

---

## 폼 설계 원칙

| 원칙 | 적용 방법 |
|------|-----------|
| **모바일 퍼스트** | 한 페이지당 질문 3~5개, 스크롤 최소화 |
| **부담 없는 진입** | 쉬운 질문부터 시작 → 어려운 질문은 뒤로 |
| **조건부 로직** | 선택에 따라 후속 질문만 노출 (불필요한 질문 숨김) |
| **기본값 제안** | 개발자 권장 옵션을 미리 선택 상태로 표시 |
| **중간 저장** | Tally 기본 기능 — 페이지 이동 시 자동 저장 |
| **진행률 표시** | 상단 프로그레스 바로 완료 비율 표시 |

## Tally.so 설정 가이드

### 폼 생성 순서

1. **Tally.so 접속** → [tally.so](https://tally.so) 무료 가입
2. **새 폼 생성** → "Create form" 클릭
3. **페이지 구분** → 각 Page를 Tally의 "Page break" 블록으로 구분
4. **프로그레스 바** → Settings > "Show progress bar" 활성화
5. **조건부 로직** → 해당 질문의 "Conditional logic" 설정 (위 표에서 "조건부"로 표시된 항목)
6. **기본값** → 확정 항목(네이버페이, 알림톡 등)은 미리 체크 상태로 설정
7. **알림 설정** → Settings > Notifications > 이메일 알림 활성화
8. **응답 연동** → Google Sheets 또는 Notion 연동 (Integrations 탭)
