# 보안 및 컴플라이언스 정책

적용된 보안 정책을 정리한 문서입니다.

---

## 1. 개인정보 보호 및 암호화 (Data Protection)

- **비밀번호 단방향 암호화:** 업계 표준인 `bcrypt` 또는 `scrypt` 알고리즘을 사용하여 비밀번호를 해싱 처리합니다. 개발자를 포함한 그 누구도 원본 비밀번호를 복호화할 수 없습니다.

- **데이터 전송 암호화 (SSL/TLS):** 모든 데이터 통신은 HTTPS 프로토콜을 통해 암호화되어 전송됩니다. 중간 취득(Sniffing)에 의한 정보 유출을 차단합니다.

- **민감 정보 분리 관리:** 개인정보보호법에 따라 주민등록번호 등 과도한 정보 수집을 배제하며, 필수적인 개인정보는 데이터베이스 내에서도 별도의 보안 계층에서 관리됩니다.


---

## 2. 인프라 및 네트워크 보안 (Infrastructure)

AWS(Amazon Web Services)를 기반으로 망 분리 시스템을 가동합니다.

- **VPC(Virtual Private Cloud) 구축:** 서버와 데이터베이스를 독립된 가상 네트워크 안에 배치하여 외부로부터의 직접적인 접근을 차단합니다.

- **보안 그룹(Security Group) 운영:** 필요한 포트(HTTPS 등) 외의 모든 접근 경로를 폐쇄하며, 관리자 전용 IP를 통해서만 시스템 접근이 가능하도록 화이트리스트 정책을 유지합니다.

- **DB 접근 제한:** 데이터베이스는 퍼블릭 인터넷과 완전히 격리된 서브넷에 배치하여, 오직 인증된 애플리케이션 서버를 통해서만 통신이 가능합니다.


---

## 3. 애플리케이션 보안 (Application Security)

코드 레벨에서 발생할 수 있는 취약점을 사전에 차단하는 보안 설계를 적용합니다.

- **인증 및 권한 관리 (JWT):** 위조가 불가능한 JSON Web Token을 사용하여 세션 탈취 위험을 낮추고, 관리자 페이지는 다중 인증(MFA) 설정을 지원합니다.

- **웹 공격 방어:** SQL Injection, XSS(Cross-Site Scripting), CSRF 등 주요 웹 보안 위협에 대비한 프레임워크 수준의 보안 필터를 기본 탑재합니다.

- **API 보안:** 모든 API 엔드포인트는 권한 검사 가드(Guard)를 통과해야 하며, 비정상적인 호출을 감지하여 차단하는 Rate Limiting을 적용합니다.


---

## 4. 안정성 및 장애 대응 (Stability)

장애는 발생하지 않는 것보다, 발생했을 때 얼마나 빠르게 복구하느냐가 중요합니다.

- **실시간 모니터링:** 서버의 CPU, 메모리 상태 및 API 응답 속도를 실시간으로 모니터링하여 이상 징후 발생 시 관리자에게 즉시 알림을 발송합니다.

- **자동 백업 시스템:** 매일 정해진 시간에 데이터베이스를 자동 백업하며, 재난 상황 발생 시에도 24시간 이내의 데이터로 즉시 복구가 가능한 환경을 구축합니다.

- **장애 탐지 및 로깅:** 모든 에러 로그는 체계적으로 기록되며, 신속한 원인 파악과 조치가 가능합니다.


---

## 5. 법적 컴플라이언스 준수

대한민국 개인정보보호법에 따른 기술적·관리적 보호 조치 기준을 준수합니다.

- **개인정보 처리방침 가이드:** 자사몰 운영에 필수적인 개인정보 처리방침 초안 작성을 지원하며, 데이터 파기 절차 등 법적 의무 사항을 시스템적으로 지원합니다.

- **데이터 최소 수집 원칙:** 마케팅에 꼭 필요한 정보만을 수집하도록 설계하여, 향후 ISMS 등 보안 인증 필요 시 대응이 용이한 구조를 유지합니다.
