## 데이터 마이그레이션 가이드

스마트스토어에 쌓인 회원 정보, 자사호·다구 상품 데이터, 구매 후기를 체계적인 ETL(추출-변환-로드) 프로세스를 통해 자사몰로 이전합니다.

---

## 1. 이전 가능한 데이터 항목

| 항목 | 상세 내용 | 비고 |
|------|----------|------|
| **회원 정보** | 이름, 이메일, 휴대전화, 주소지, 적립금 현황 | 비밀번호는 보안상 재설정 필요 |
| **상품 데이터** | 자사호·다구 상품명, 옵션(용량/작가), 가격, 재고, 상세 설명, 고해상도 이미지 | 카테고리 구조 재설계 포함 |
| **리뷰/게시글** | 구매 리뷰, 별점 (자사호 사용 후기 등), Q&A 게시글 | 텍스트 및 첨부 이미지 전체 |
| **주문 이력** | 과거 주문 내역, 배송 상태 (선택 사항) | 통계 분석용 데이터로 활용 가능 |

---

## 2. 데이터 이전 4단계 프로세스

### 1단계: 스마트스토어 데이터 추출 (Extraction)

- 옥화당 스마트스토어 관리자 페이지에서 상품·주문·회원 데이터를 엑셀(CSV)로 직접 추출합니다.
- 유실된 이미지가 없는지, 텍스트가 깨진 부분은 없는지 1차 전수 조사를 실시합니다.

### 2단계: 데이터 매핑 및 변환 (Transformation)

- 스마트스토어의 구조를 자사몰 고성능 DB 구조(MySQL 8.0 + TypeORM 마이그레이션)에 맞게 재배열합니다.
- 중복 가입자 처리, 휴먼 계정 분류, 자사호·다구 카테고리 체계 최적화 작업을 수행합니다.

### 3단계: 테스트 서버 업로드 (Mock Migration)

- 실제 오픈 전, 개발 서버에 전체 데이터를 미리 업로드하여 정합성을 테스트합니다.
- 링크 연결 오류, 이미지 로딩 속도, 옵션 선택 정상 작동 여부를 검증합니다.

### 4단계: 최종 마이그레이션 (Final Cutover)

- 자사몰 오픈 직전, 최신 데이터를 최종 반영합니다.
- **무중단 이전(Zero-downtime):** 서비스 중단 시간을 최소화하여 고객 불편을 방지합니다.

---

## 3. 기술적 난제 해결

- **자사호 고해상도 이미지 최적화:** 옥화당은 작가 작품 사진, 디테일 샷 등 고해상도 이미지가 특히 많습니다. 수천 장의 이미지를 옮기면서도 사이트 속도가 느려지지 않도록 **WebP 변환 및 CDN 캐싱** 작업을 병행합니다.

- **비밀번호 보안 정책:** 비밀번호는 암호화되어 있어 직접 이전이 불가능합니다. 기존 회원이 로그인 시 비밀번호를 재설정하거나, 간편 로그인(카카오/구글)으로 연동할 수 있도록 **UX 시나리오를 설계**합니다.

- **SEO 주소 유지:** 기존 스마트스토어 상세 페이지의 URL이 바뀌어도 검색 엔진에서 "페이지를 찾을 수 없음"이 뜨지 않도록 **301 리다이렉트** 설정을 지원합니다.

## 브랜드사가 준비해주실 사항

성공적인 이사를 위해 아래 자료를 준비해 주시면 즉시 분석에 들어갑니다.

1. 옥화당 스마트스토어 관리자 권한 (상품·주문·회원 데이터 추출용)
2. 이전 대상 우선순위 확정 (예: 상품 데이터·회원 데이터는 필수, 3년 전 주문 내역은 선택 등)
3. 회원 대상 '비밀번호 재설정 및 이전 안내' 공지 문구 (가이드 제공 가능)
