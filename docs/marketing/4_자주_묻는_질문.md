## 1. 프로젝트 진행 및 소통

**중간에 연락이 안 되거나 일정이 지연되면 어쩌죠?**

> 주 1회 정기적인 진행 상황 공유를 원칙으로 합니다. 기획·디자인·개발 각 단계별로 고객의 확인과 승인을 거친 후 다음 단계로 넘어가며, 중간 산출물을 투명하게 공개하여 리스크를 최소화합니다. 카카오톡·슬랙으로 실시간 소통 채널을 운영하며, 모든 과정은 문서화됩니다.

**완성된 결과물이 생각했던 디자인과 다르면 어쩌죠?**

> 본격적인 개발 착수 전, 와이어프레임과 디자인 시안을 먼저 공유합니다. 이 단계에서 충분한 피드백을 반영하여 확정된 디자인을 바탕으로 개발에 들어갑니다. 시안 단계에서 방향이 맞지 않으면 개발 착수 전에 수정하므로, 최종 결과물에서 발생하는 이견을 사전에 차단합니다.

**오픈 후에 버그가 발생하면 어떻게 대응해 주시나요?**

> 납품 후 **1개월간 무상으로 버그 및 결함에 대응**해 드립니다. 장애 등급에 따라 다음과 같이 대응합니다:
> - **긴급 (서비스 중단):** 2시간 이내 확인, 8시간 이내 복구
> - **일반 (기능 오류):** 24시간 이내 확인, 72시간 이내 해결
> - **경미 (UI 오류):** 다음 정기 패치 시 처리

## 2. 1인 개발자 리스크

**개발자 한 명이 아프거나 사정이 생기면 프로젝트가 멈추지 않나요?**

> 1인 개발 리스크에 대한 대비책:
> - **단계별 중간 산출물 공유:** 매 단계 완료 시 고객이 직접 확인할 수 있는 결과물을 제공하므로, 어떤 상황에서도 진행 상황이 고객 손에 있습니다.
> - **표준화된 코드 구조:** 업계 표준(Next.js + NestJS)으로 작성하므로, 불가피한 상황 발생 시 다른 개발자가 즉시 이어받을 수 있습니다.
> - **운영 문서 납품:** 서버 설정, API 명세, 배포 가이드를 함께 납품하여 의존성을 최소화합니다.

**나중에 저와의 계약이 종료되어도 다른 개발자에게 맡길 수 있나요?**

> 네, 가능합니다. 특정 개인만 이해할 수 있는 코드가 아닌, **업계 표준 기술(Next.js, NestJS 등)**과 깔끔한 코드 구조로 제작합니다. 서버 설정법, API 명세서 등이 담긴 운영 문서를 함께 납품하므로, 실력 있는 개발자라면 누구나 즉시 업무를 이어받을 수 있습니다.

## 3. 기술적 독립성 및 유지보수

**결제사(PG)나 택배사를 나중에 바꾸고 싶으면 전면 재개발해야 하나요?**

> 아닙니다. 처음부터 확장을 고려한 '어댑터 패턴'으로 설계합니다. 토스페이먼츠에서 나이스페이로, 자체 배송에서 외부 풀필먼트로 전환하더라도 어드민 설정과 간단한 연결만으로 교체가 가능합니다.

**사내 개발자가 직접 코드를 수정해도 되나요?**

> 네, 납품 시 소스코드의 전권과 서버 관리자 권한을 이전해 드립니다. 표준화된 개발 환경이므로 기능 추가나 수정에 제약이 없습니다.

## 3. 기술적 독립성 및 유지보수

**개인정보 보호나 보안 사고가 걱정됩니다.**

> 모든 비밀번호는 bcrypt 해시 암호화로 복호화 불가 처리하며, JWT 기반 보안 인증 체계를 사용합니다. AWS 보안 그룹 설정으로 비정상 접근을 차단하고, 개인정보보호법 준수 설계로 안전한 쇼핑 환경을 구축합니다. Rate Limiting으로 비정상적인 API 호출도 자동 차단합니다.

**이벤트 날 트래픽이 몰려서 사이트가 터지면 어쩌죠?**

> B마트 커머스에서의 대규모 트래픽 처리 경험을 기반으로 설계. 애플리케이션 레이어 in-memory 캐싱과 이미지 최적화(CDN)로 반복 조회 부하를 줄이며, 트래픽 급증 시 AWS에서 서버 사양을 즉시 확장할 수 있는 유연한 인프라 구조를 제공합니다. Vercel Edge Network가 프론트엔드 부하를 자동으로 처리합니다.

## 4. 데이터 및 마케팅 (SEO)

**옥화당이 현재 운영 중인 스마트스토어의 상품·고객 데이터를 자사몰로 이전할 수 있나요?**

> 네, 스마트스토어에서 추출한 상품·회원 데이터를 자사몰 구조에 맞게 변환하여 일괄 등록하는 마이그레이션 작업을 지원합니다. 자사호 상품의 작가명, 용량, 산지, 재질 등 세부 속성도 그대로 옮겨집니다. (데이터의 양과 복잡도에 따라 일정과 비용은 협의 후 결정됩니다.)

**자사호 같은 고가 상품은 상세 설명과 사진이 매우 중요한데, 자사몰에서는 어떻게 더 잘 보여줄 수 있나요?**

> 스마트스토어는 템플릿 구조상 레이아웃과 표현 방식에 제약이 많습니다. 자사몰에서는 그 한계 없이, 풀스크린 이미지 갤러리·작가 스토리·산지(이싱) 정보·유약 설명 등 옥화당만의 브랜드 콘텐츠를 자유롭게 구성할 수 있습니다.

**기존 스마트스토어와 자사몰을 병행 운영할 수 있나요?**

> 네, 동시 운영이 가능합니다. 스마트스토어를 바로 닫지 않고 자사몰과 함께 운영하면서, 단골 고객과 신규 유입을 자사몰로 점진적으로 전환할 수 있습니다. 자사몰 안착 후 스마트스토어 비중을 줄여 나가는 방식이 일반적이며, 그 타이밍은 옥화당의 판단에 따라 자유롭게 조절하실 수 있습니다.

**구글이나 네이버 검색 노출(SEO)이 정말 잘 되나요?**

> 기술적으로 보장합니다. SSR(서버 사이드 렌더링)과 구조화 데이터(JSON-LD)를 적용하여, 검색 엔진 로봇이 상품의 별점·가격·재고 정보를 정확히 읽어가게 합니다. 검색 결과에서 상품 정보가 풍부하게 표시되어 클릭률이 높아집니다.

**우리 고객 데이터를 마케팅에 직접 활용할 수 있나요?**

> 브랜드 소유의 DB를 가집니다. 특정 상품을 장바구니에 담은 고객에게만 쿠폰 알림톡을 보내는 등 데이터 기반의 정교한 마케팅이 가능해집니다.

## 5. 비용 및 소유권

**소스코드와 DB의 최종 소유권은 누구에게 있나요?**

> 잔금 완납 시 소스코드·DB·서버 소유권이 고객사로 귀속됩니다.

**서버 유지비가 매달 많이 나오면 어쩌죠?**

> 월 약 15만 원입니다 (Vercel Pro + AWS EC2 + Lightsail DB + S3/CloudFront 포함). 매출이 커질수록 플랫폼 수수료 대비 서버 고정비는 훨씬 저렴해집니다.

**기능을 하나 추가할 때마다 비용이 계속 불어나나요?**

> 초기 계약 시 구현 범위를 명확히 확정하여 불필요한 추가 견적 발생을 차단합니다. 납품 후 새로운 기능이 필요한 경우에도, 표준화된 코드 기반이므로 합리적인 단위 기능 비용으로 확장이 가능합니다.
