# Commerce Demo — 서비스 기획서

다양한 쇼핑몰 외주에 재사용 가능한 D2C 자사몰 보일러플레이트.

---

## 프로젝트 개요

**Commerce Demo**는 자사몰(D2C) 형태의 쇼핑몰 보일러플레이트입니다. 실제 PG 결제 연동과 배송 추적을 포함한 실무 수준의 완성된 구조를 갖추어, 새 프로젝트 시작 시 즉시 포크하여 커스터마이징할 수 있는 기반을 목표로 합니다.

> ChaLog 프로젝트와 기술 스택/구현 방향성을 통일합니다.

---

## 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| **ChaLog 통일** | 기술 스택, 코드 스타일, Git 워크플로우, 배포 구조를 ChaLog와 맞춤 |
| **템플릿 기반** | 모든 UI 조형 요소는 하드코딩이 아닌 템플릿으로 관리. 어드민에서 템플릿 선택/설정 가능 (토스 방식 참조) |
| **DB 드리븐 UI** | 카테고리, 사이드바, 네비게이션 등 프론트 구조를 DB로 관리하고 어드민에서 CRUD |
| **반응형 우선** | 모든 화면이 데스크톱/모바일에서 깨지지 않는 flex 기반 UI |
| **보안 우선** | 어드민, 인증, PG 연동 등 전 영역에 보안 고려 |
| **Docker 기반 인프라** | MySQL, Redis 등 모든 서비스를 Docker Compose로 통합 관리 |
| **재사용성** | 외주 프로젝트마다 포크하여 빠르게 커스터마이징 가능한 구조 |

## 주요 기능 목록

### 상품 탐색
- 메인 페이지 (히어로 배너, 추천/인기 상품, 카테고리 바로가기)
- 카테고리별 상품 목록 (그리드/리스트 뷰, 정렬, 필터)
- 상품 상세 (이미지 갤러리, 옵션, 리뷰, 문의 탭)
- 상품 검색 (자동완성, 최근/인기 검색어)

### 구매 프로세스
- 장바구니 (추가/삭제/수량 변경, 합계 계산)
- 주문/결제 (배송지 입력, PG 결제, 쿠폰/적립금 적용)
- 주문 완료 및 주문 내역 조회

### PG 결제
- 어댑터 패턴 기반 PG 추상화 (토스페이먼츠, KG이니시스 등)
- 개발: Mock PG / 운영: 실제 PG
- 서버 사이드 금액 검증 필수
- 결제 상태 머신 (대기 → 승인 → 취소/환불)

### 배송 시스템
- 배송 상태 관리 (결제완료 → 상품준비중 → 배송중 → 배송완료)
- 택배사 API 추상화 (CJ대한통운, 한진, 롯데 등 어댑터 패턴)
- 운송장 등록 및 실시간 배송 추적

### 회원
- 회원가입 (이메일/비밀번호)
- 로그인/로그아웃 + OAuth (카카오/구글)
- 마이페이지 (주문내역, 회원정보, 배송지 관리)

### 커뮤니케이션
- 상품 리뷰 (별점, 포토 리뷰)
- 위시리스트 (찜하기)
- 쿠폰/적립금
- 공지사항, FAQ, 1:1 문의

### 어드민
- 페이지/템플릿 관리 (블록 구성, 설정값 편집)
- 네비게이션 관리 (GNB, 사이드바, 카테고리 CRUD)
- 상품/주문/배송/회원 관리
- 쿠폰/프로모션 관리
- 매출/주문/방문 통계 대시보드 (recharts)

## 화면 구성 / 페이지 맵

### 공개 화면 (비로그인 접근 가능)

| 경로 | 화면명 | 설명 |
|------|--------|------|
| `/` | 메인 | 히어로 배너, 추천/인기 상품, 카테고리 바로가기 |
| `/products` | 상품 목록 | 카테고리별 리스트, 필터/정렬 |
| `/products/:id` | 상품 상세 | 이미지 갤러리, 옵션, 리뷰, 문의 |
| `/search` | 검색 결과 | 검색어 기반 상품 목록 |
| `/cart` | 장바구니 | 상품 목록, 수량 변경, 합계 |
| `/login` | 로그인 | 이메일 / 카카오 / 구글 |
| `/register` | 회원가입 | 이메일 회원가입 |
| `/notice` | 공지사항 | 공지 리스트/상세 |
| `/faq` | FAQ | 카테고리별 아코디언 |
| `/event` | 이벤트 | 기획전, 타임세일 |

### 로그인 필요 화면

| 경로 | 화면명 | 설명 |
|------|--------|------|
| `/checkout` | 주문/결제 | 배송지, 결제, 쿠폰 적용 |
| `/order/complete` | 주문 완료 | 주문 확인 |
| `/mypage` | 마이페이지 | 개요 |
| `/mypage/orders` | 주문 내역 | 주문 리스트 |
| `/mypage/orders/:id` | 주문 상세 | 상태 타임라인, 배송 추적 |
| `/mypage/wishlist` | 위시리스트 | 찜한 상품 |
| `/mypage/profile` | 회원정보 수정 | 프로필 편집 |
| `/mypage/address` | 배송지 관리 | 배송지 CRUD |
| `/mypage/coupons` | 쿠폰 | 보유 쿠폰 목록 |

### 관리자 화면 (`/admin`, role: admin 전용)

| 경로 | 설명 |
|------|------|
| `/admin/dashboard` | 대시보드: 매출/주문/방문 통계 |
| `/admin/products` | 상품 관리: CRUD, 옵션/재고, 노출 상태 |
| `/admin/orders` | 주문 관리: 상태 변경, 검색/필터 |
| `/admin/shipping` | 배송 관리: 운송장 등록, 상태 변경 |
| `/admin/members` | 회원 관리: 조회, 역할 변경 |
| `/admin/coupons` | 쿠폰/프로모션 관리 |
| `/admin/pages` | 페이지/템플릿 관리 |
| `/admin/navigation` | 네비게이션 관리 |

## 구현 우선순위

```
Phase 1 (MVP)     : 메인, 상품 목록/상세, 장바구니, 주문/결제(Mock PG) — 핵심 구매 플로우
Phase 2 (Core)    : 회원 시스템, 검색, 카테고리 필터, 마이페이지
Phase 3 (Payment) : 실제 PG 연동, 배송 시스템 구축
Phase 4 (Admin)   : 어드민 (상품/주문/배송/회원 관리)
Phase 5 (CMS)     : 템플릿 시스템, 동적 페이지/네비게이션 관리
Phase 6 (Polish)  : 리뷰, 위시리스트, 쿠폰, 성능 최적화
Phase 7 (Ops)     : CI/CD, 모니터링, 테스트 커버리지
```
